# Requirements — Tab "ObjectProperties"

> Generated by reverse-engineering of the SWOWL source code · version 1.0 · 2026-06-06

---

## 1. Navigation et sélection dans l'arbre

- **REQ-OP-001** — Display object properties as a hierarchical tree in the left panel
  - *If* If the ObjectProperties tab is displayed, then the tree is built from subPropertyOf relationships in the application state

- **REQ-OP-002** — Sort tree nodes in case-insensitive alphabetical order
  - *If* If multiple properties share the same level, then they are sorted alphabetically (localeCompare, base sensitivity)

- **REQ-OP-003** — Display the root node owl:topObjectProperty at the top of the tree, always expanded
  - *If* If the tab is rendered, then the root displays the label from APP.getOntologyRootLabels().propRoot

- **REQ-OP-004** — Select a property by clicking its node in the tree
  - *If* If the user clicks a node, then that node receives the CSS class 'selected', the detail panel is updated and the Super Properties panel is refreshed

- **REQ-OP-005** — Select the owl:topObjectProperty root node by clicking on it
  - *If* If the user clicks the root, then _topPropSelected is set to true, _selectedId to null, and the detail panel shows an invitation message with a create button

- **REQ-OP-006** — Preserve selection and expanded node state after section reload
  - *If* If the section is reloaded (restoreSelection), then the previously selected property is re-selected and ancestors are expanded

- **REQ-OP-007** — Expand or collapse a node with children by clicking the toggle arrow
  - *If* If the node is open and the user clicks ▶, then the subtree is hidden and the 'open' class is removed from the toggle; and vice versa

- **REQ-OP-008** — Display a leaf indicator (◦) for nodes without children
  - *If* If a node has no children in childrenOf, then a ◦ symbol is displayed instead of the toggle

- **REQ-OP-009** — Navigate to a property from another tab via APP.navigateTo
  - *If* If APP.navigateTo('object-properties', id) is called, then the target property is selected and its ancestors are expanded (_expandAncestors)

- **REQ-OP-010** — Display a ↔ tag on the node if the property has a declared inverse
  - *If* If prop.inverseOf is non-null, then a badge 'op-inverse-tag' showing '↔ {inverseOf}' is added to the node label in the tree

---

## 2. Création de propriétés

- **REQ-OP-011** — Create a child property of the selected property via the 'Child' button
  - *If* If the user clicks 'Child', then a new property is created with subPropertyOf containing the selected property ID (or empty if root is selected)

- **REQ-OP-012** — Create a sibling property at the same level as the selected property via the 'Sibling' button
  - *If* If the user clicks 'Sibling', then a new property is created with the same parents (subPropertyOf) as the selected property

- **REQ-OP-013** — Automatically generate a unique identifier of the form 'NewObjectProperty' or 'NewObjectPropertyN'
  - *If* If the name 'NewObjectProperty' already exists, then an incremental integer suffix is appended until a free identifier is found

- **REQ-OP-014** — Call POST /api/object-properties to persist the new property
  - *If* If creation fails with HTTP 409, then an error is displayed indicating the identifier already exists

- **REQ-OP-015** — Validate that the identifier is a valid NCName before creation or update
  - *If* If the identifier does not pass NCName validation on the backend (_validate_ncname), then an HTTPException 400 is raised

- **REQ-OP-016** — Automatically open the edit form in 'new' mode after creation
  - *If* If the property is successfully created, then _editingId is set to the new identifier and the section is reloaded with the property selected

---

## 3. Edition du formulaire de détail

- **REQ-OP-017** — Display the Protégé-style form in the right panel when a property is selected
  - *If* If a property is selected, then the detail panel displays the ID, full IRI, annotations, domain, range, inverse and characteristics

- **REQ-OP-018** — Display the full IRI of the property in the form header (baseIRI#localId)
  - *If* If the property has an identifier and the ontology has a base IRI, then the full IRI is displayed in the form baseIRI#localId

- **REQ-OP-019** — Sanitize invalid characters in the ID field in real time (oninput)
  - *If* If the user types an unauthorized character in the ID field, then _sanitizeId is called to replace it

- **REQ-OP-020** — Automatically save the form on every field change (autoSave / onchange)
  - *If* If _editingId is non-null and a field changes, then OPEditor.autoSave() calls save(false) and PUT /api/object-properties/{id}

- **REQ-OP-021** — Save a new property when the ID field loses focus (onblur)
  - *If* If the property is new (isNew=true) and the ID field loses focus with a non-empty value, then save(true) is called and POST /api/object-properties is emitted

- **REQ-OP-022** — Rename the property when the identifier changes and cascade the rename
  - *If* If the submitted ID differs from the original ID, then the backend runs _cascade_rename_property to update all references (subPropertyOf, inverseOf, etc.)

- **REQ-OP-023** — Manage rdfs:label, rdfs:comment and custom annotation properties
  - *If* If the user clicks '+ label' or '+ comment', then a new row is added in the annotation table with default value and language

- **REQ-OP-024** — Add custom annotations via an annotation property picker
  - *If* If the user clicks '+ Annotation Property', then a picker listing available annotation properties is toggled

- **REQ-OP-025** — Remove an annotation row via its ✕ button
  - *If* If the user clicks ✕ on an annotation, then the row is removed from the DOM and autoSave is triggered

- **REQ-OP-026** — Manage property domain: add and remove OWL classes via a class tree picker
  - *If* If no domain is defined, then 'owl:Thing' is shown as the implicit value; if the user adds a class, it is added to the list and autoSave is triggered

- **REQ-OP-027** — Manage property range: add and remove OWL classes via a class tree picker
  - *If* If no range is defined, then 'owl:Thing' is shown as the implicit value; if the user adds a class, it is added to the list and autoSave is triggered

- **REQ-OP-028** — Synchronize domain markers on OWL classes during create, update and delete operations
  - *If* If a property domain is modified, then _sync_domain_markers is called on the backend to update added and removed classes

## 4. Gestion de la relation inverse (inverseOf)

- **REQ-OP-029** — Set the inverse property via a picker that filters out properties already inverse of another
  - *If* If the user opens the inverse picker, then properties that already have an inverseOf pointing to a third property are excluded from the list

- **REQ-OP-030** — Allow only one inverse property (uniqueness): the + button is hidden once an inverse is defined
  - *If* If op-inverse-value contains a non-empty value, then the add inverse button is hidden and the picker does not open

- **REQ-OP-031** — Maintain owl:inverseOf symmetry on the backend during create, update and delete
  - *If* If property A declares inverseOf = B, then the backend automatically sets B.inverseOf = A; and if A is deleted, B.inverseOf is reset to None

- **REQ-OP-032** — Display inverses inferred by the inference engine below the Inverse Of section
  - *If* If the inference engine returns inferred_inverse_properties for the selected property, then '⊢ inverse of X' badges are shown with the inference reason as tooltip

- **REQ-OP-033** — Allow navigation to the inverse property by clicking its label in the form
  - *If* If the user clicks the inverse property label in the detail panel, then APP.navigateTo('object-properties', inverseId) is called

## 5. Caractéristiques OWL de la propriété

- **REQ-OP-034** — Display and edit the 7 OWL characteristics via checkboxes: Functional, InverseFunctional, Transitive, Symmetric, Asymmetric, Reflexive, Irreflexive
  - *If* If the user checks or unchecks a characteristic, then autoSave is triggered and the property is updated via PUT /api/object-properties/{id}

- **REQ-OP-035** — Persist characteristics in the 'characteristics' field (key/boolean object) of OWLObjectProperty

## 6. Super Properties (panneau gauche bas)

- **REQ-OP-036** — Display the 'Super Properties' panel in the lower left column, linked to the selected property
  - *If* If no property is selected, then the panel shows '— select a property —'; otherwise it lists the super-properties of the selection

- **REQ-OP-037** — Display the full ancestry chain of each super-property up to owl:topObjectProperty
  - *If* If super-property P1 has its own parent P2, then P2 is displayed below P1 with increasing indentation and reduced opacity

- **REQ-OP-038** — Add a super-property via a tree picker in the Super Properties panel
  - *If* If the user selects a property in the picker, then it is added to subPropertyOf and autoSave is triggered

- **REQ-OP-039** — Remove a direct super-property via the ✕ button on its row in the panel
  - *If* If the user clicks ✕ on a direct super-property, then it is removed from subPropertyOf and autoSave is triggered

- **REQ-OP-040** — Navigate to a super-property by clicking its label in the panel
  - *If* If the user clicks a super-property label (direct or ancestor), then APP.navigateTo('object-properties', id) is called

- **REQ-OP-041** — Navigate to owl:topObjectProperty by clicking its entry in the Super Properties panel
  - *If* If the user clicks the owl:topObjectProperty entry in the panel, then OPEditor.selectTopProp() is called

---

## 7. Drag & Drop pour réorganiser la hiérarchie

- **REQ-OP-042** — Allow moving a property by drag and drop onto another tree node
  - *If* If the user drops property A onto property B, then A's subPropertyOf is replaced by [B] and PUT /api/object-properties/A is called

- **REQ-OP-043** — Allow moving to the owl:topObjectProperty root (empty subPropertyOf)
  - *If* If the user drops a property on the root node, then the property's subPropertyOf is reset to []

- **REQ-OP-044** — Forbid dropping onto a descendant to prevent cycles
  - *If* If the drop target is a descendant of the dragged property, then the operation is cancelled and a warning 'Cannot drop on a descendant' is displayed

- **REQ-OP-045** — Apply visual feedback styles during drag (class 'dragging' on source, 'drag-over' on target)
  - *If* If drag starts, then the 'dragging' class is added to the source; if the cursor hovers over a valid target, 'drag-over' is added and removed on leave

---

## 8. Suppression de propriété

- **REQ-OP-046** — Delete the selected property via the Delete button in the action bar
  - *If* If the user clicks Delete and confirms the modal dialog, then DELETE /api/object-properties/{id} is called and the selection is reset

- **REQ-OP-047** — Ask for explicit confirmation before any deletion
  - *If* If the user triggers deletion, then UI.confirm shows 'Delete ObjectProperty <strong>{id}</strong>?'; if the user cancels, no action is taken

- **REQ-OP-048** — Remove inverseOf symmetry on the backend when deleting a property that has an inverse
  - *If* If the deleted property has inverseOf = B and B.inverseOf = A, then B.inverseOf is set to None before A is deleted

---

## 9. Menu contextuel et boutons d'action

- **REQ-OP-049** — Display a context menu on right-click on a tree node
  - *If* If the user right-clicks a node, then a context menu appears with 'Add Child Property', 'Add Sibling Property' and 'Delete' options (the last two absent for the root)

- **REQ-OP-050** — Position the context menu to avoid overflow beyond window edges
  - *If* If the menu overflows the right or bottom edge of the window, then its coordinates are adjusted to remain within the visible area

- **REQ-OP-051** — Close the context menu when clicking outside of it
  - *If* If a click is detected outside the context menu, then it is removed from the DOM

- **REQ-OP-052** — Enable/disable 'Child', 'Sibling' and 'Delete' buttons based on selection context
  - *If* If the root is selected, then only 'Child' is active; if a property is selected, all three buttons are active; if nothing is selected, all are disabled

---

## 10. Mise en page et redimensionnement

- **REQ-OP-053** — Allow horizontal resizing of the left panel via a draggable split handle
  - *If* If the user drags the vertical split handle, then the left panel width is adjusted between 160 px and 520 px

- **REQ-OP-054** — Allow vertical resizing of the tree / Super Properties area in the left column
  - *If* If the user drags the internal horizontal resizer (h-resizer), then the relative height of the two sub-panels is changed

- **REQ-OP-055** — Allow vertical resizing of the detail form sections via internal separators
  - *If* If a property is selected and the form is displayed, then _initHResizers('op-detail') is called to activate the vertical separators in the right panel

- **REQ-OP-056** — Display an empty state with a create button when no property is selected
  - *If* If neither the root nor a property is selected, then the right panel shows 'Select an existing Object Property or create a new one' with a '＋ Create Object Property' button

## 11. Interactions backend (API REST)

- **REQ-OP-057** — List all object properties via GET /api/object-properties when the tab loads

- **REQ-OP-058** — Return HTTP 404 if a non-existent property is accessed, updated or deleted
  - *If* If prop_id does not match any property in onto.object_properties, then HTTPException 404 is raised

- **REQ-OP-059** — Return HTTP 409 if a duplicate identifier is submitted during creation or rename
  - *If* If prop.id already exists for another record, then HTTPException 409 is raised

- **REQ-OP-060** — Persist ontology state to disk after every create, update or delete operation
  - *If* If a write operation succeeds, then store.save() is called on the backend

- **REQ-OP-061** — Display a success message (UI.success) or error message (UI.error) based on API call results
  - *If* If the API returns an error, then UI.error(e.message) is called; if the operation succeeds, a success toast is displayed

---

## 12. Utilisation dans les règles SWRL (Where Used)

- **REQ-OP-062** — Display a 'Where Used' frame at the bottom of the form listing SWRL rules that reference the property
  - *If* If SWRL rules use the selected property (_ruleUsesProperty), then they are listed in the _whereUsedFrame
