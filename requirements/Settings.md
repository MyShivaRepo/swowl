# Requirements — Tab "Settings"

> Generated by reverse-engineering of the SWOWL source code · version 1.0 · 2026-06-06

---

## 1. Navigation et structure de l'onglet Settings

- **REQ-SET-001** — The Settings tab is accessible from the main navigation bar via a dedicated menu item.
  - *If* If the user clicks the 'Settings' nav item, then the main content area renders the Settings panel.

- **REQ-SET-002** — The Settings tab is a fixed (required) tab: it cannot be hidden from the GUI tabs manager.
  - *If* If a tab is marked as 'fixed', then its checkbox is disabled and it displays the 'required' badge.

- **REQ-SET-003** — The Settings panel is organized into two areas: a vertical navigation sidebar and a main content area.

- **REQ-SET-004** — The sidebar contains three sub-tabs: 'GUI Tabs', 'Languages', and 'IDs Rules'.

- **REQ-SET-005** — The active sub-tab is visually highlighted: left accent border, bold colored text, distinct background.
  - *If* If a sub-tab is active, then its left border is accent-colored, its text is font-weight 600 in accent color, and its background is 'bg3'.

- **REQ-SET-006** — Clicking a sidebar sub-tab updates the internal '_settingsTab' state and re-renders the Settings section.
  - *If* If the user clicks a sub-tab, then APP._settingsTab is updated and APP.renderSection('settings') is called.

- **REQ-SET-007** — The last active sub-tab is remembered in the '_settingsTab' state variable (default: 'gui-tabs').
  - *If* If no sub-tab has been selected, then 'gui-tabs' is displayed by default.

---

## 2. Gestion de la visibilité des onglets GUI (sous-onglet 'GUI Tabs')

- **REQ-SET-008** — The 'GUI Tabs' sub-tab displays the full list of application tabs with their icon and label.

- **REQ-SET-009** — Optional tabs are: AnnotationProperties, Individuals, SWRL Rules, Views, Queries, Inferences. They can be hidden or shown.

- **REQ-SET-010** — Required tabs (Ontologies, Settings, Classes, ObjectProperties, DatatypeProperties) are displayed with reduced opacity and are not clickable.
  - *If* If a tab is 'fixed', then its checkbox is disabled, cursor is 'default', and opacity is 0.55.

- **REQ-SET-011** — Clicking an optional tab in the list toggles its visibility in the navigation bar.
  - *If* If the user clicks an optional tab row, then TabVisibility.toggle() is called for that tab.

- **REQ-SET-012** — The visibility state of each optional tab is persisted in localStorage under the key 'swowl_hidden_tabs'.
  - *If* If an optional tab's visibility changes, then the hidden tabs array is serialized and saved to localStorage.

- **REQ-SET-013** — Hiding the tab that the user is currently on redirects to the 'Ontologies' tab.
  - *If* If the hidden tab is the current section (APP.currentSection), then the application navigates to 'ontologies'.

- **REQ-SET-014** — Changing a tab's visibility immediately updates the navigation bar display (without page reload).
  - *If* If a tab's visibility changes, then APP._applyTabVisibility() is called to update the navigation bar DOM.

- **REQ-SET-015** — When a tab's visibility changes from the 'GUI Tabs' sub-tab, the Settings content is immediately re-rendered to reflect the checkbox state.
  - *If* If APP._settingsTab === 'gui-tabs' and a tab changes visibility, then APP.renderSection('settings') is called.

- **REQ-SET-016** — Optional tab rows have a hover effect that changes their background color.
  - *If* If the user hovers an optional tab row, then the background changes to 'var(--bg3)'; on mouse leave it returns to 'var(--bg2)'.

## 3. Gestion des langues (sous-onglet 'Languages')

- **REQ-SET-017** — The 'Languages' sub-tab presents three distinct sections: preferred language, active languages, available languages.

- **REQ-SET-018** — The available languages list covers 25 European languages (from bg/Bulgarian to sv/Swedish).

- **REQ-SET-019** — The preferred language is the one applied by default to new rdfs:label and rdfs:comment. It must belong to active languages.
  - *If* If a language is set as preferred, then it is automatically added to active languages if not already present.

- **REQ-SET-020** — The 'Preferred language' section displays one button per active language; the preferred language is highlighted with a filled star (★) and 'primary' style.
  - *If* If a language is the preferred language, then its button uses 'btn-primary' style and displays '★'; otherwise it uses 'btn-secondary' and displays '☆'.

- **REQ-SET-021** — Clicking a language button in the 'Preferred language' section sets that language as preferred and re-renders the Settings panel.
  - *If* If the user clicks a preferred language button, then Settings.setPreferred(code) is called, state is saved, and the section is re-rendered.

- **REQ-SET-022** — The 'Active languages' section displays active languages as badges with language code and a remove button (✕), except for the preferred language.
  - *If* If an active language is not the preferred language, then it displays a ✕ button allowing its removal.

- **REQ-SET-023** — The preferred language cannot be removed from the active languages.
  - *If* If the user tries to deactivate the preferred language via toggleActive(), then the action is ignored (immediate return).

- **REQ-SET-024** — The 'Available languages' section displays all 25 European languages as buttons; active languages are highlighted with 'primary' style.
  - *If* If a language is in the active languages list, then its button uses 'btn-primary' style; otherwise 'btn-secondary'.

- **REQ-SET-025** — Clicking an available language button toggles its active/inactive status and re-renders the Settings panel.
  - *If* If the language is already active, then it is removed from active languages; otherwise it is added.

- **REQ-SET-026** — Language settings (preferred language and active languages) are persisted in localStorage under the key 'swowl_settings'.
  - *If* If language settings are changed, then Settings.save() serializes preferredLang, activeLangs and namingFormat to localStorage.

- **REQ-SET-027** — On application load, if the preferred language is not in the active languages, it is automatically added at the start of the list.
  - *If* If on load activeLangs does not contain preferredLang, then preferredLang is inserted at position 0 of activeLangs.

- **REQ-SET-028** — The preferred language serves as the default language for the LANG dropdown selector in the editing forms of other tabs.
  - *If* If Settings.preferredLang is defined, then it is used as the initial value of the 'language' attribute of rdfs:label and rdfs:comment components.

## 4. Règles de nommage des Individuals (sous-onglet 'IDs Rules')

- **REQ-SET-029** — The 'IDs Rules' sub-tab allows configuring the ID format automatically generated when creating a new Individual.

- **REQ-SET-030** — Three naming formats are offered: 'Individual_Counter', 'ClassName_Counter', and 'Alphanumeric string'.

- **REQ-SET-031** — The 'Individual_Counter' format generates an ID of the form 'Individual_N' where N is the count of existing individuals + 1.
  - *If* If the format is 'individual_counter', then the generated ID is 'Individual_' followed by the total count of existing individuals + 1.

- **REQ-SET-032** — The 'ClassName_Counter' format generates an ID of the form '{ClassId}_N' where ClassId is the selected class ID and N is the counter.
  - *If* If the format is 'class_counter' and a class is selected, then the ID is '{classId}_{counter}'; otherwise it falls back to 'Individual_N'.

- **REQ-SET-033** — The 'Alphanumeric string' format generates a random ID of 4 groups of 5 alphanumeric characters separated by dashes.
  - *If* If the format is 'alphanumeric', then the ID is generated with the pattern '{seg1}-{seg2}-{seg3}-{seg4}' where the first segment starts with a letter and contains 4 additional alphanumeric characters.

- **REQ-SET-034** — Each format option is displayed with its label, a description, and a concrete example of the generated identifier.

- **REQ-SET-035** — The selected format is visually highlighted: accent border and 'bg3' background.
  - *If* If a format option is selected, then its border is 'var(--accent)' and background is 'var(--bg3)'; otherwise border is 'var(--border)' and background is 'var(--bg2)'.

- **REQ-SET-036** — Selecting a format via radio button triggers Settings.setNamingFormat() which saves and re-renders the Settings panel.
  - *If* If the user changes the naming format, then Settings.namingFormat is updated, Settings.save() is called, and APP.renderSection('settings') is executed.

- **REQ-SET-037** — The naming format is persisted in localStorage and reloaded on application startup (default value: 'individual_counter').
  - *If* If no format is found in localStorage, then the value 'individual_counter' is used as default.

- **REQ-SET-038** — The automatically generated ID can be manually edited by the user before confirming the creation of the Individual.

## 5. Persistance et chargement des paramètres

- **REQ-SET-039** — All user settings (preferred language, active languages, naming format) are stored in a single JSON object in localStorage.
  - *If* If Settings.save() is called, then the object {preferredLang, activeLangs, namingFormat} is serialized under the key 'swowl_settings' in localStorage.

- **REQ-SET-040** — Settings are loaded from localStorage on application startup, before the first render.
  - *If* If Settings.load() is executed, then localStorage values override the default values of the Settings object.

- **REQ-SET-041** — If localStorage contains corrupted or invalid data, settings silently fall back to default values.
  - *If* If JSON.parse() throws an exception during loading, then default values are kept without any error message.

- **REQ-SET-042** — The visibility of optional tabs is managed in a separate persistence object (key 'swowl_hidden_tabs') distinct from general settings.

---

## 6. Comportements visuels et interactions générales

- **REQ-SET-043** — The Settings panel displays a '🛠️ Settings' title in a header bar separated from the content by a border.

- **REQ-SET-044** — The language selection dropdown (LANG) in other tabs positions itself below the trigger button and lists active languages.
  - *If* If Settings.showLangDropdown() is called, then a fixed-position floating menu is shown below the button, listing active languages.

- **REQ-SET-045** — The selected language in the LANG dropdown is visually highlighted (accent background, white text).
  - *If* If a dropdown item matches the current input value, then its background is 'var(--accent)' and its text is white.

- **REQ-SET-046** — The LANG dropdown closes automatically when the user clicks outside of it.
  - *If* If a click is detected outside the dropdown and the trigger button, then the dropdown is removed from the DOM.

- **REQ-SET-047** — Selecting a language in the LANG dropdown updates the associated input value and triggers a 'change' event.
  - *If* If the user clicks a LANG dropdown item, then inp.value is updated and a 'change' event with bubbles:true is dispatched.

- **REQ-SET-048** — Only one LANG dropdown can be open at a time: opening a new dropdown closes any existing dropdown.
  - *If* If Settings.showLangDropdown() is called, then document.getElementById('lang-dropdown')?.remove() is executed first.
