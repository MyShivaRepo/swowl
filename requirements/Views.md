# Requirements — Tab "Views"

> Generated by reverse-engineering of the SWOWL source code · version 1.0 · 2026-06-06

---

## 1. Structure générale et navigation dans l'onglet

- **REQ-VIEW-001** — The Views tab is optional and can be hidden from the Settings (GUI Tabs) panel.
  - *If* If the user unchecks 'Views' in Settings > GUI Tabs, then the Views tab is removed from the navigation bar.

- **REQ-VIEW-002** — The Views tab displays a left sidebar with two sub-tabs: 'Ontology' and 'Knowledge Base'.

- **REQ-VIEW-003** — The active sub-tab is highlighted with a colored left border (accent color), bold font weight, and a distinct background.
  - *If* If a sub-tab is active, then its left border is colored with the accent color and its background is var(--bg3).

- **REQ-VIEW-004** — Clicking a sidebar sub-tab updates the internal state APP._viewsTab and re-renders the Views section.
  - *If* If the user clicks 'Ontology' or 'Knowledge Base', then APP._viewsTab is updated and APP.renderSection('views') is called.

- **REQ-VIEW-005** — The graph corresponding to the selected sub-tab is initialized asynchronously (setTimeout 80 ms) after HTML rendering.
  - *If* If the active sub-tab is 'ontology', then APP._initHyperbolicGraph() is called 80 ms after rendering. If it is 'knowledge-base', then APP._initKnowledgeBase() is called.

- **REQ-VIEW-006** — The default sub-tab value is 'ontology' if no value has been previously stored.

## 2. Sous-onglet Ontology — Arbre hyperbolique (Disque de Poincaré)

- **REQ-VIEW-007** — The Ontology sub-tab displays a hyperbolic tree graph (Poincaré Disk) of all ontology classes using D3.js.

- **REQ-VIEW-008** — If D3.js is not loaded, an error message is displayed instead of the graph.
  - *If* If typeof d3 === 'undefined', then the container displays '⚠ D3.js not loaded — check network connection.'

- **REQ-VIEW-009** — If the ontology contains no classes, an informational message is displayed instead of the graph.
  - *If* If APP.state.classes is empty, then the message 'No classes in this ontology.' is displayed.

- **REQ-VIEW-010** — The virtual root 'owl:Thing' is automatically added as the tree root; classes without an internal parent become its direct children.
  - *If* If a class has no parent within the ontology's class set, then it is attached to owl:Thing.

- **REQ-VIEW-011** — Node positions are computed using Poincaré hyperbolic geometry via Möbius transformations.

- **REQ-VIEW-012** — A node's visual appearance (radius, opacity, font size, color) varies according to its distance from the center of the disk.
  - *If* If a node is near the center (dist < 0.35), then its circle is larger, more opaque, and its label more visible. If a node is at the periphery (dist > 0.78), then its label is hidden.

- **REQ-VIEW-013** — A node's label is determined first by the rdfs:label annotation in the preferred language, then by the class identifier.
  - *If* If a class has an rdfs:label in the preferred language, then that label is used. Otherwise, the raw class identifier is displayed.

- **REQ-VIEW-014** — A single click on a peripheral node moves it to the center of the disk via a Möbius transformation, with a 0.42 s CSS animation.
  - *If* If the user clicks a node with dist >= 0.10, then mobiusFocus is applied to bring it to the center and _hypDraw(true) is called.

- **REQ-VIEW-015** — Clicking a node already at the center (dist < 0.10, excluding owl:Thing) navigates to the class editor and selects that class.
  - *If* If a node is at the center (dist < 0.10) and is not owl:Thing, then APP.navigate('classes') is called and ClassEditor._selectedId is set to that node's identifier.

- **REQ-VIEW-016** — The 'Reset' button restores all nodes to their initial layout positions, with animation.
  - *If* If the user clicks '⟳ Reset', then each node reverts to its basePos and _hypDraw(true) is called.

- **REQ-VIEW-017** — A text filter field allows searching for classes by label or identifier; matching nodes are highlighted in green.
  - *If* If the user types in 'Filter classes…', then nodes whose label or identifier contains the query (case-insensitive) have their circle stroke set to '#10b981' and label to '#6ee7b7'. Other nodes revert to normal appearance.

- **REQ-VIEW-018** — A counter displays the total number of classes in the ontology in the graph toolbar.

- **REQ-VIEW-019** — The hyperbolic disk background is a dark SVG circle with concentric guide rings at 33%, 60%, and 82% of the radius.

- **REQ-VIEW-020** — Edges between parent and child nodes are SVG lines that animate in color and opacity alongside nodes.

- **REQ-VIEW-021** — The SVG canvas size is dynamically adapted to the container size; if the height is insufficient, it is computed from window.innerHeight.
  - *If* If container.offsetHeight < 20, then the height is set to window.innerHeight - 170.

- **REQ-VIEW-022** — A hint text 'Click → focus · Double-click → edit' is displayed in the toolbar to guide the user.

## 3. Sous-onglet Knowledge Base — Graphe de force (Individuals)

- **REQ-VIEW-023** — The 'Knowledge Base' sub-tab displays a D3 force-directed graph representing all ontology individuals and their relationships via object property assertions.

- **REQ-VIEW-024** — If no individuals are present in the ontology, an informational message is displayed.
  - *If* If APP.state.individuals is empty, then 'Aucun individual dans cette ontologie.' is displayed.

- **REQ-VIEW-025** — Each individual is represented by a node colored according to its class (class_id or first type), with a label above and the class name below.

- **REQ-VIEW-026** — Class colors are automatically assigned from a palette of 15 predefined colors, stored in APP._kbColorMap.
  - *If* If a class has not yet been assigned a color, then the next available color in the palette is assigned cyclically.

- **REQ-VIEW-027** — Edges represent object property assertions between individuals; they carry a label showing the property name.
  - *If* If an individual has objectAssertions whose target is a known individual, then an edge is created between the two nodes.

- **REQ-VIEW-028** — Edges are directional and display an arrowhead SVG marker at their target end.

- **REQ-VIEW-029** — The force simulation uses repulsion between nodes, link attraction, centering, and collision forces to prevent overlapping.

- **REQ-VIEW-030** — The user can freely move a node via drag-and-drop; the simulation briefly restarts during the drag.
  - *If* If the user starts dragging a node, then alphaTarget is set to 0.3 and the simulation restarts. When drag ends, alphaTarget returns to 0 and fx/fy constraints are released.

- **REQ-VIEW-031** — On node hover (mouseover), unconnected nodes and edges are dimmed (opacity reduced to 0.05–0.2) to highlight immediate neighbors.
  - *If* If the user hovers over a node, then non-neighbor nodes go to opacity 0.2 and unconnected edges go to opacity 0.05. On mouseout, everything returns to opacity 1.

- **REQ-VIEW-032** — Clicking an individual node navigates to the individuals editor and selects that individual.
  - *If* If the user clicks a KB graph node, then APP.navigate('individuals') is called and IndividualEditor._selectedId is set to that individual's identifier.

- **REQ-VIEW-033** — The '⟳ Restart' button relaunches the force simulation with an alpha of 0.8 to redistribute nodes.
  - *If* If the user clicks '⟳ Restart' and a simulation is active, then APP._kbSim.alpha(0.8).restart() is called.

- **REQ-VIEW-034** — When reinitializing the KB graph, the previous simulation is stopped and color mappings are reset.
  - *If* If APP._initKnowledgeBase() is called while a simulation exists, then APP._kbSim.stop() is called, APP._kbColorMap and APP._kbColorIndex are reset.

- **REQ-VIEW-035** — A text filter field allows searching for individuals by label or class identifier; non-matching nodes are dimmed to opacity 0.1.
  - *If* If the user types in 'Filter individuals…', then nodes whose label or classId contains the query remain at opacity 1; others go to 0.1.

- **REQ-VIEW-036** — A color legend lists the classes represented in the graph, with a color dot and the name of each class.

- **REQ-VIEW-037** — A counter displays the number of individuals and the number of connections (edges) in the KB graph toolbar.

- **REQ-VIEW-038** — The KB graph supports zoom and pan via d3.zoom, with a scale range from 0.1× to 4×.

- **REQ-VIEW-039** — An individual's label in the KB graph is determined first by the labels annotation in the preferred language, then by its identifier.
  - *If* If an individual has a labels annotation in the preferred language, then that label is used. Otherwise, the raw identifier is displayed.

- **REQ-VIEW-040** — Nodes are initialized with random positions around the canvas center to avoid initial overlap.

## 4. Comportements visuels et retours d'interface

- **REQ-VIEW-041** — Hyperbolic graph animation transitions use a cubic-bezier(0.33, 1, 0.68, 1) curve over 0.42 s.
  - *If* If animated=true is passed to _hypDraw, then CSS transitions are enabled. If animated=false, no transition is applied.

- **REQ-VIEW-042** — The owl:Thing root node is rendered with a distinctive style: radius 11 px, fill '#1e3a5f', and blue stroke '#3b82f6'.
  - *If* If a node has the identifier 'owl:Thing', then it is displayed with radius 11 px, fill '#1e3a5f', and stroke '#3b82f6'.

- **REQ-VIEW-043** — Hyperbolic node labels are displayed in bold for nodes very close to the center (dist < 0.15).
  - *If* If dist < 0.15, then font-weight is '600'. Otherwise, font-weight is '400'.

- **REQ-VIEW-044** — In the KB graph, edges use two arrowhead markers: a normal (gray) one and a highlighted (blue) one, used according to hover state.

- **REQ-VIEW-045** — The Ontology graph background is '#12161f' and the KB graph background is '#0e1219', creating a visual distinction between the two views.

## 5. Intégration et navigation inter-onglets

- **REQ-VIEW-046** — Clicking a centered class node in the Ontology graph navigates directly to the Classes tab and selects the relevant class in the editor.
  - *If* If the node is at the center (dist < 0.10) and is not owl:Thing, then APP.navigate('classes') is called and ClassEditor._selectedId is set with an 80 ms delay.

- **REQ-VIEW-047** — Clicking an individual node in the KB graph navigates to the Individuals tab and selects the relevant individual in the editor.
  - *If* If the user clicks a KB node, then APP.navigate('individuals') is called and IndividualEditor._selectedId is set with a 120 ms delay.

- **REQ-VIEW-048** — The _whereUsedFrame function in owl_editor.js displays, in editor panels (classes, properties, individuals), the SWRL rules that use the edited entity.
  - *If* If an entity (class, property, or individual) is referenced in at least one SWRL rule, then a 'Where Used in Rules' panel is displayed listing those rules.

- **REQ-VIEW-049** — In the 'Where Used in Rules' panel, clicking a rule navigates to the SWRL Rules tab and selects that rule via APP.navigateTo('swrl-rules', id).
  - *If* If the user clicks a rule in the 'Where Used' panel, then APP.navigateTo('swrl-rules', r.id) is called.

- **REQ-VIEW-050** — In the 'Where Used in Rules' panel, the rule label is displayed preferentially, with the technical identifier as subtext if a label exists.
  - *If* If r.label is defined, then mainText = r.label and subText = r.id. Otherwise, mainText = r.id and subText is empty.

- **REQ-VIEW-051** — In the 'Where Used in Rules' panel, the rule label is underlined and changes color (accent) on mouse hover.
  - *If* If the user hovers over a rule label in 'Where Used', then text-decoration becomes 'underline' and color becomes 'var(--accent)'.
