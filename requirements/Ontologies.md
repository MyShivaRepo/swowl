# Requirements — Tab "Ontologies"

> Generated by reverse-engineering of the SWOWL source code · version 1.0 · 2026-06-06

---

## 1. Navigation et affichage de l'onglet

- **REQ-ONT-001** — The Ontologies tab is the first tab opened by default when the application starts.
  - *If* If no ontology is connected at launch, then the Ontologies tab is displayed.

- **REQ-ONT-002** — Switching to the Ontologies tab is recorded in the navigation history (◀ ► buttons).
  - *If* If the user navigates to the Ontologies tab, then the previous state is pushed onto the history stack and the ◀ ► buttons are updated.

- **REQ-ONT-003** — If a hidden optional tab is the active tab, navigation is automatically redirected to the Ontologies tab.
  - *If* If a tab is hidden while it is active, then the application automatically navigates to the Ontologies tab.

- **REQ-ONT-004** — The Ontologies tab displays a section header titled « Ontologies ».

- **REQ-ONT-005** — When the Ontologies tab is rendered, the registry table data is loaded asynchronously from the backend.
  - *If* If loading fails, then the table row displays « Unable to load the registry. ».

- **REQ-ONT-006** — The ontology counter in the registry bar displays the total number of ontologies present.
  - *If* If the list contains N ontologies, then the displayed text is « N ontology » (singular) or « N ontologies » (plural).

---

## 2. Table du registre

- **REQ-ONT-007** — The registry table displays columns: connection indicator, name, directory, prefix, namespace (URI), and actions.

- **REQ-ONT-008** — Ontologies are sorted in the table: user ontologies first (alphabetical order), then W3C built-in ontologies (OWL, RDFS, RDF) last.
  - *If* If an ontology is marked readonly, then it is displayed after all user ontologies, in order OWL > RDFS > RDF.

- **REQ-ONT-009** — The currently connected ontology is visually highlighted in the table (CSS class onto-current-row) and its indicator shows a green dot ●.
  - *If* If an ontology is connected, then its indicator is ● green; otherwise it is ○ grey.

- **REQ-ONT-010** — Clicking on a table row selects the corresponding ontology (highlighted via the onto-selected-row CSS class).
  - *If* If the user clicks on a row, then the previously selected row loses its highlight and the new row is highlighted.

- **REQ-ONT-011** — The « Directory » cell is clickable and triggers revealing the file in Finder (via host_agent.py).
  - *If* If host_agent.py is not running, then a warning « Start host_agent.py to enable Finder reveal. » is displayed.

- **REQ-ONT-012** — W3C (readonly) ontologies display a « W3C » label and a lock icon 🔒 instead of edit and delete buttons.
  - *If* If an ontology is marked readonly, then only the Connect/Disconnect button is shown (no ✏️, ✕, or export buttons).

- **REQ-ONT-013** — If the table contains no ontologies, an invitation message is displayed instead of rows.
  - *If* If the registry list is empty, then the message « No ontologies yet — use a button above to get started. » is displayed.

- **REQ-ONT-014** — Auto-selection of the row is performed on load: if no manual selection exists, the connected ontology is automatically selected.
  - *If* If no ontology is manually selected and an ontology is connected, then that ontology is pre-selected in the table.

---

## 3. Arbre des imports (sous-lignes)

- **REQ-ONT-015** — Each user ontology displays its OWL imports as nested sub-rows within the table.
  - *If* If an ontology declares no explicit imports, then the implicit import http://www.w3.org/2002/07/owl# is displayed.

- **REQ-ONT-016** — Import sub-rows display a ▶/▼ toggle if the import has its own sub-imports, or an arrow ↳ otherwise.
  - *If* If the user clicks on the ▶ toggle, then the sub-imports of that import are displayed and the toggle changes to ▼.

- **REQ-ONT-017** — The expansion state of import sub-rows is remembered within the session (set _ontoImportExpanded).
  - *If* If the user expanded an import node, then that node remains expanded when the table is refreshed (without an additional API call).

- **REQ-ONT-018** — Clicking on the name of a resolved import (present in the registry) scrolls to and flashes the corresponding main row in the table.
  - *If* If the clicked import is resolved in the registry, then the row is scrolled into view and an accent outline flashes for 1.5 seconds.

- **REQ-ONT-019** — Import cycles are detected and interrupted to avoid infinite recursion when rendering sub-rows.
  - *If* If an import URI is already present in the visited ancestor chain, then it is not rendered again.

---

## 4. Boutons d'action du registre (par ontologie)

- **REQ-ONT-020** — The « ▶ Connect » button connects the selected ontology and makes it active for all other tabs.
  - *If* If the connection succeeds, then a success toast is displayed, the global state is reloaded, and the table is refreshed.

- **REQ-ONT-021** — The « ⏏ Disconnect » button disconnects the active ontology.
  - *If* If disconnection succeeds, then the editing tabs display the message « No ontology connected » with a link to the Ontologies tab.

- **REQ-ONT-022** — The « ✏️ » button opens the wizard panel in edit mode pre-filled with the ontology metadata (name, directory, prefix, URI).
  - *If* If the same type of wizard is already open, then clicking this button again replaces its content with the edit form.

- **REQ-ONT-023** — The « ↓ Ontology » button opens a dropdown menu allowing export of the ontology in OWL (.owl) or Turtle (.ttl) format.
  - *If* If the user selects an export format, then the file is downloaded directly in the browser under the name {name}.owl or {name}.ttl.

- **REQ-ONT-024** — The « ↓ Rules » button opens a dropdown menu allowing export of SWRL rules in JSON (.json) or SWORD (.sword) format.
  - *If* If the user selects SWRL JSON, then the ontology rules are serialized and downloaded as {name}_rules.swrl.json.

- **REQ-ONT-025** — The « ✕ » button removes the ontology from the registry after user confirmation, without deleting the file on disk.
  - *If* If the user cancels the confirmation dialog, then the ontology remains in the registry.

- **REQ-ONT-026** — Export dropdown menus close automatically if the user clicks outside of them.

---

## 5. Wizards de création / import / chargement

- **REQ-ONT-027** — The Ontologies tab displays four main action buttons: « ✨ New Ontology », « 📥 Import Ontology », « 📂 Load Ontology », and « W3C Fetch W3C Ontologies ».

- **REQ-ONT-028** — Clicking an action button opens the corresponding wizard panel; a second click on the same button closes the panel (toggle behavior).
  - *If* If the wizard panel is already open with the same type, then it closes. Otherwise it opens with the corresponding content.

- **REQ-ONT-029** — The « New Ontology » wizard collects: name (required), directory (required, selectable via file browser), prefix (default: onto), and namespace URI (required).
  - *If* If any required field is empty, then a toast error message is displayed and creation is cancelled.

- **REQ-ONT-030** — The « New Ontology » wizard offers a « Connect immediately » checkbox checked by default to connect the ontology right after creation.
  - *If* If the checkbox is checked, then the ontology is connected immediately after being registered.

- **REQ-ONT-031** — The « Import Ontology » wizard accepts source files in .owl, .ttl, .rdf, and .xml formats.
  - *If* If the selected file is in Turtle format (.ttl), then the backend parses it with the « turtle » format; otherwise the « xml » format is used.

- **REQ-ONT-032** — The « 🔍 Read Prefix & URI from file » button in the Import wizard analyzes the source file and automatically pre-fills the prefix and URI fields.
  - *If* If the prefix and URI fields are already filled, then only the empty field is updated by auto-detection.

- **REQ-ONT-033** — The « Load Ontology » wizard accepts only .json files (SWOWL internal format).
  - *If* If the 🔍 button is clicked without a file selected, then a warning toast is displayed.

- **REQ-ONT-034** — The « Load Ontology » wizard offers the « 🔍 Read info from file » button that automatically pre-fills the name, prefix, and URI from the JSON file.

- **REQ-ONT-035** — All wizards offer a « Cancel » button that closes the panel without modifying the registry.

- **REQ-ONT-036** — After each successful wizard operation (create, import, load, edit), the wizard is closed, the global state is reloaded, and the registry table is refreshed.

---

## 6. Navigateur de fichiers (FsBrowser)

- **REQ-ONT-037** — File or directory selection fields in wizards open a modal file browser integrated into the application.
  - *If* If the user clicks on the readonly field, then the FsBrowser modal opens on the remembered current directory or /Users/bernard by default.

- **REQ-ONT-038** — The file browser displays a clickable breadcrumb allowing quick navigation to a parent directory.
  - *If* If the user clicks on a breadcrumb segment, then the list content is reloaded for that directory.

- **REQ-ONT-039** — The file browser operates in two modes: file selection (filtered extensions) and directory selection.
  - *If* In file mode, the « Select this file » button is disabled until a file is clicked. In directory mode, the « Select this folder » button is always active.

- **REQ-ONT-040** — A double-click on a file in the browser selects it and immediately closes the modal, confirming the selection.

- **REQ-ONT-041** — The file browser modal closes if the user clicks outside the modal window.

- **REQ-ONT-042** — If a directory is not accessible (Docker permissions), the browser displays an explanatory error message.
  - *If* If the error contains « 403 » or « Permission », then the message « Permission denied — this folder is not accessible from Docker. » is displayed.

---

## 7. Fetch des ontologies W3C intégrées

- **REQ-ONT-043** — The « W3C Fetch W3C Ontologies » button downloads the three core W3C ontologies (RDF, RDFS, OWL) from w3.org and registers them as read-only in the registry.
  - *If* If the download succeeds, then the number of fetched ontologies is displayed in a success toast and the table is refreshed.

- **REQ-ONT-044** — The « W3C Fetch W3C Ontologies » button is disabled during the download to prevent double submissions.
  - *If* If the download is in progress, then the button is disabled (disabled=true) and is re-enabled at the end (success or error).

- **REQ-ONT-045** — Downloaded W3C ontologies are stored in ~/.swowl/builtins/ and registered as read-only in the registry.

---

## 8. Édition des métadonnées d'une ontologie

- **REQ-ONT-046** — The ontology edit form allows modifying its name, directory, prefix, and namespace URI.
  - *If* If the name, directory, or URI is empty at save time, then a toast error is displayed and the save is cancelled.

- **REQ-ONT-047** — Modifying the name or directory of an ontology automatically recalculates the file path (.json) from the entered directory and name.
  - *If* On save, the path is constructed as dir + '/' + name + '.json' stripping any trailing '/' from the directory.

- **REQ-ONT-048** — The backend checks the uniqueness of the ontology name in the registry on creation and returns an HTTP 409 error if the name already exists.
  - *If* If a duplicate name is submitted, then the backend returns 409 and the frontend displays the corresponding error message.

---

## 9. Gestion de l'état de connexion et impact sur les autres onglets

- **REQ-ONT-049** — When no ontology is connected, the editing tabs (Classes, Properties, Individuals, SWRL, Views, Queries, Inferences) display a blocking message with a link to the Ontologies tab.
  - *If* If APP.state.ontology is null and the requested section is an editing tab, then the content is replaced by the « No ontology connected » message.

- **REQ-ONT-050** — After connecting or disconnecting an ontology, the global application state (classes, properties, individuals, rules) is reloaded from the backend.

- **REQ-ONT-051** — The prefix of the connected ontology determines the virtual root labels displayed in the editors (owl:Thing / rdfs:Resource, owl:topObjectProperty / rdf:Property).
  - *If* If the ontology prefix is « rdf » or « rdfs », then the roots are rdfs:Resource and rdf:Property; otherwise they are owl:Thing and owl:topObjectProperty.

---

## 10. Notifications et retours utilisateur

- **REQ-ONT-052** — All registry operations (create, import, load, edit, connect, disconnect, delete) result in a success or error toast displayed for 3.5 seconds.

- **REQ-ONT-053** — Removing an ontology from the registry is preceded by a modal confirmation dialog.
  - *If* If the user clicks outside the confirmation modal, then the action is cancelled.

- **REQ-ONT-054** — The delete confirmation dialog specifies that the file on disk will not be deleted.
