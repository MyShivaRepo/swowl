# Requirements — Tab "DatatypeProperties"

> Generated by reverse-engineering of the SWOWL source code · version 1.0 · 2026-06-06

---

## 1. Navigation et sélection

- **REQ-DP-001** — Display the tab in a two-column split layout (tree on the left, detail on the right) separated by a horizontally resizable divider
  - *If* If the user drags the vertical split handle, then the left panel width is adjusted between 160 px and 520 px

- **REQ-DP-002** — Display the hierarchy root (owl:topDataProperty or rdf:Property depending on the ontology) at the top of the tree, always visible and non-deletable
  - *If* If the ontology root is rdf:Property, then the displayed root node is 'rdf:Property', otherwise it is 'owl:topDataProperty'

- **REQ-DP-003** — Select a property in the tree by clicking its node: the row is highlighted and the detail form is shown in the right panel
  - *If* If the user clicks a tree node, then that node receives the CSS class 'selected' and the corresponding form is rendered in #dp-detail

- **REQ-DP-004** — Selecting the root node (owl:topDataProperty) displays an empty detail panel with a property creation button
  - *If* If the user clicks the root node, then the right panel shows the root label, a 'Root of all Datatype Properties' subtitle, and the '＋ Create Datatype Property' button

- **REQ-DP-005** — Restore the previous selection on every section re-render (after a save or refresh)
  - *If* If _topPropSelected is true, then selectTopProp() is re-called; else if _selectedId is set, then selectProp(_selectedId) is re-called

- **REQ-DP-006** — Display a 'No DatatypeProperty' message in the tree when the ontology contains no datatype property
  - *If* If the datatype properties list is empty, then display the text 'No DatatypeProperty' instead of tree nodes

- **REQ-DP-007** — Navigate directly to a datatype property from another tab via APP.navigateTo('datatype-properties', id)
  - *If* If APP.navigateTo is called with section 'datatype-properties' and an id, then ancestors are expanded and the property is automatically selected

## 2. Arbre hiérarchique

- **REQ-DP-008** — Build and display the property tree respecting subPropertyOf relations: properties without a parent are roots, others are children
  - *If* If a property A has subPropertyOf = [B], then A is displayed as a child of B in the tree

- **REQ-DP-009** — Sort nodes at each tree level in case-insensitive alphabetical order

- **REQ-DP-010** — Display a clickable ▶ triangle for nodes with children, and a ◦ dot for leaf nodes
  - *If* If a node has at least one child, then display the 'tree-toggle' control; otherwise display 'tree-leaf'

- **REQ-DP-011** — Collapse/expand subtrees by clicking the ▶ triangle of a parent node (without changing the current selection)
  - *If* If a node's toggle is clicked, then the node's expanded state is toggled and the child container display is updated

- **REQ-DP-012** — Visually indent each tree level by 16 px increments

## 3. Création et suppression

- **REQ-DP-013** — Create a child property of the selected property via the 'Child' toolbar button or the context menu
  - *If* If createChild() is called with a property selected, then the new property has subPropertyOf = [selectedId] and the parent is expanded in the tree

- **REQ-DP-014** — Create a sibling property (same level as the selected property) via the 'Sibling' toolbar button or the context menu
  - *If* If createSibling() is called, then the new property inherits the same parents as the currently selected property

- **REQ-DP-015** — Automatically generate a unique identifier in the form 'NewDatatypeProperty', 'NewDatatypeProperty1', etc. upon creation
  - *If* If 'NewDatatypeProperty' already exists, then increment the numeric suffix until a free identifier is found

- **REQ-DP-016** — Disable the 'Sibling' and 'Delete' buttons when the root node is selected; enable only the 'Child' button
  - *If* If _topPropSelected is true, then btnSister and btnDelete are disabled and hidden, and btnChild is enabled

- **REQ-DP-017** — Ask for confirmation before deleting a property, displaying its identifier in the message
  - *If* If deleteSelected() is called, then show UI.confirm('Delete DatatypeProperty <strong>id</strong>?') and only execute deletion upon confirmation

- **REQ-DP-018** — After deletion, reset the selection (_selectedId = null) and refresh the full section display

- **REQ-DP-019** — Reject creation if an identical identifier already exists (HTTP 409 returned by the backend)
  - *If* If POST /api/datatype-properties receives an already existing id, then return HTTP 409 with message 'DatatypeProperty id already exists'

## 4. Formulaire de détail (édition)

- **REQ-DP-020** — Display and allow editing of the property's local identifier (IRI) in a text field with real-time sanitization
  - *If* If the user modifies the #dp-id field, then _sanitizeId() is called on each keystroke to strip invalid characters

- **REQ-DP-021** — Display the full property IRI (baseIRI#localName) in read-only mode below the identifier field
  - *If* If the property has an identifier and the ontology has a base IRI, then display 'For Property: baseIRI#localName'

- **REQ-DP-022** — Validate the identifier client-side: reject an empty value or one starting with a digit (OWL NCName rule)
  - *If* If the identifier is empty or starts with a digit, then display an error and cancel the save

- **REQ-DP-023** — Automatically save form changes (auto-save) as soon as a field changes, without explicit user action
  - *If* If _editingId is not null and a form field triggers onchange, then autoSave() calls save(false) immediately

- **REQ-DP-024** — Rename a property: if the identifier is changed, cascade the change across all ontology references (subPropertyOf, assertions, SWRL rules)
  - *If* If PUT /api/datatype-properties/{old_id} receives a different new id, then _cascade_rename_property() is called to update all references

- **REQ-DP-025** — Manage rdfs:label and rdfs:comment annotations: add rows, edit value and language, individual deletion
  - *If* If the user clicks '+ label' or '+ comment', then a new row is added to the annotations table with the default language

- **REQ-DP-026** — Allow adding custom annotations (Annotation Property) via a dedicated picker listing available annotation properties
  - *If* If the user clicks '+ Annotation Property', then the picker #dp-anno-picker is toggled

- **REQ-DP-027** — Manage the property domain(s): select one or more OWL classes via a dropdown, individually remove each entry
  - *If* If no domain is defined, then display 'owl:Thing' as default value; if a domain is added, then the presence marker is synchronized in the concerned classes

- **REQ-DP-028** — Manage the XSD type (range) of the property: select a single type from the 12 supported XSD types via a picker
  - *If* If no range is defined, then display 'rdfs:Literal' as default value; the range '+' button is hidden once a type is selected

- **REQ-DP-029** — Show the range '+' button again after the selected XSD type is removed
  - *If* If removeRange() is called, then button #dp-range-btn is made visible

- **REQ-DP-030** — Manage the 'Functional' characteristic via a checkbox: a functional property can have only one value per individual
  - *If* If the 'Functional' checkbox is checked/unchecked, then the functional field is updated and auto-save is triggered

- **REQ-DP-031** — Display a 'Where Used in Rules' section listing all SWRL rules referencing the property, with navigation links to each rule
  - *If* If at least one SWRL rule references the property in its body or head atoms, then the 'Where Used in Rules' frame is displayed with the rule count and clickable links

## 5. Panneau Super Properties

- **REQ-DP-032** — Display the 'Super Properties' panel in the left column (below the tree), separated by a vertically resizable divider
  - *If* If the user drags the horizontal resizer between the tree and the Super Properties panel, then the relative heights of both areas are adjusted

- **REQ-DP-033** — Display the full inheritance chain up to owl:topDatatypeProperty for each direct super-property, with progressive indentation
  - *If* If property A has super-property B, and B has super-property C, then the list displays A (direct), then B (ancestor, opacity 0.75), then owl:topDatatypeProperty (root, italic, opacity 0.55)

- **REQ-DP-034** — Add a super-property via the '+' button in the Super Properties panel: opens a picker listing available properties (excluding the property itself and already selected ones)
  - *If* If a super-property is selected in the picker, then addSubProp(id) is called and auto-save is triggered

- **REQ-DP-035** — Remove a direct super-property via the ✕ button displayed next to its name in the panel
  - *If* If a super-property's ✕ button is clicked, then removeSubProp(id) is called and auto-save is triggered

- **REQ-DP-036** — Allow navigation to an ancestor super-property by clicking its label in the panel
  - *If* If the user clicks an ancestor's label, then APP.navigateTo('datatype-properties', id) is called

- **REQ-DP-037** — Display '— select a property —' in the Super Properties panel when no property is selected

## 6. Drag & Drop

- **REQ-DP-038** — Allow moving a property in the hierarchy by drag-and-drop onto a target node in the tree
  - *If* If a property is dropped onto a target node, then its subPropertyOf list is replaced by [targetId] and the change is saved via PUT /api/datatype-properties/{id}

- **REQ-DP-039** — Prevent dropping a property onto one of its own descendants to avoid cycles in the hierarchy
  - *If* If the drop target is a descendant of the dragged property, then show UI.warn('Cannot drop on a descendant — would create a cycle') and cancel the operation

- **REQ-DP-040** — Prevent dropping a property onto itself
  - *If* If dragId === targetId, then the drop operation is ignored

- **REQ-DP-041** — Dropping a property onto the root node moves it to root level (removes all its parents)
  - *If* If a property is dropped onto owl:topDataProperty (targetId = null), then subPropertyOf is set to []

- **REQ-DP-042** — Apply visual styles during drag: 'dragging' class on the dragged element, 'drag-over' class on the hovered target
  - *If* If the drag starts, then 'dragging' is added via setTimeout(0); if the cursor hovers a valid target, then 'drag-over' is added to that target

- **REQ-DP-043** — Clean up all drag styles at the end of the operation (onDragEnd)
  - *If* If the dragend event fires, then the 'dragging' class is removed from the source element and 'drag-over' is removed from all elements

## 7. Menu contextuel

- **REQ-DP-044** — Display a context menu on right-click on a tree node with actions: 'Add Child Property', 'Add Sibling Property', 'Delete'
  - *If* If the right-click targets a property node (non-root), then the context menu contains all three options; if the target is the root node, then only 'Add Child Property' is available

- **REQ-DP-045** — Position the context menu at cursor position, automatically adjusting to avoid window overflow
  - *If* If the menu overflows the right or bottom edge of the window, then its position is recalculated to keep it visible

- **REQ-DP-046** — Automatically close the context menu when clicking outside it
  - *If* If a click occurs outside the #dp-ctx-menu, then _closeContextMenu() is called and the event listener is removed

## 8. Interactions backend (API REST)

- **REQ-DP-047** — List all datatype properties via GET /api/datatype-properties when the tab is loaded

- **REQ-DP-048** — Create a datatype property via POST /api/datatype-properties with the full model (id, annotations, domain, range, subPropertyOf, functional)
  - *If* If creation succeeds (HTTP 201), then UI.success displays 'DatatypeProperty id created' and the section is refreshed

- **REQ-DP-049** — Update a datatype property via PUT /api/datatype-properties/{prop_id} with the updated full model
  - *If* If the id changed during update, then UI.success displays 'Property renamed → newId'

- **REQ-DP-050** — Delete a datatype property via DELETE /api/datatype-properties/{prop_id}
  - *If* If deletion succeeds, then UI.success displays 'DatatypeProperty id deleted'

- **REQ-DP-051** — Synchronize property presence markers in classes (PropertyPresence) on any domain change (create, update, delete)
  - *If* If a property's domain changes, then _sync_domain_markers() adds or removes markers in the concerned classes

- **REQ-DP-052** — Validate the identifier server-side (OWL NCName rule): reject any empty id or one starting with a digit with HTTP 422
  - *If* If _validate_ncname() detects an invalid id, then HTTP 422 is returned with an explanatory message

- **REQ-DP-053** — Display an error message via UI.error() if a backend operation fails (network, 409, 422, 404)
  - *If* If the API call throws an exception, then UI.error(e.message) is called and no local state update is performed

## 9. Types XSD supportés

- **REQ-DP-054** — Offer exactly 12 XSD types in the range picker: xsd:string, xsd:integer, xsd:decimal, xsd:float, xsd:double, xsd:boolean, xsd:date, xsd:dateTime, xsd:duration, xsd:anyURI, xsd:nonNegativeInteger, xsd:positiveInteger
  - *If* If a type is already selected as the range, then it no longer appears in the picker

- **REQ-DP-055** — Limit the range to a single XSD value: the '+' button is hidden after selecting a first type
  - *If* If #dp-range-list already contains an item with data-id, then showPicker('dp-range-picker') does nothing

## 10. Intégration avec d'autres onglets

- **REQ-DP-056** — Allow quick creation of a datatype property with a pre-filled domain from the Classes tab (dedicated button on a class)
  - *If* If the user clicks 'Create DatatypeProperty with domain = className' from the Classes tab, then DPEditor._selectedId and _editingId are pre-initialized and the datatype-properties section is rendered

- **REQ-DP-057** — Allow navigation from the Classes view to an existing datatype property by clicking its name in the class property list
  - *If* If a property in the Classes view is a DatatypeProperty (identified by APP.state.datatype_properties), then the click navigates to the 'datatype-properties' tab and selects the property

- **REQ-DP-058** — Allow navigation from the Super Properties panel to an ancestor property by clicking its label, without leaving the DatatypeProperties tab
  - *If* If the user clicks an ancestor label in the Super Properties panel, then APP.navigateTo('datatype-properties', id) is called
