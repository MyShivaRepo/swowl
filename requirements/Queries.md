# Requirements — Tab "Queries"

> Generated by reverse-engineering of the SWOWL source code · version 1.0 · 2026-06-06

---

## 1. Navigation et structure de l'onglet

- **REQ-QRY-001** — The Queries tab is divided into two sub-tabs accessible via a vertical sidebar navigation.
  - *If* If the user clicks a sidebar tab, then the main content is reloaded with the selected sub-tab.

- **REQ-QRY-002** — The first sub-tab is labeled 'Sparnatural' and displays the visual Sparnatural component.

- **REQ-QRY-003** — The second sub-tab is labeled 'SPARQL VizQ' and displays the visual SPARQL query editor (SparqlEditor).

- **REQ-QRY-004** — The active tab is highlighted with a colored left border accent and bold font.
  - *If* If a tab is active, then its left border shows the accent color and its text is bold.

- **REQ-QRY-005** — The selected sub-tab state is persisted in the APP._queriesTab variable between renders.
  - *If* If the user switches sub-tabs and returns to the Queries tab, then the last visited sub-tab is restored.

---

## 2. Sous-onglet Sparnatural

- **REQ-QRY-006** — The Sparnatural component is loaded dynamically from an external CDN (jsdelivr.net, version 9.1.6).
  - *If* If the 'spar-natural' custom element is not registered within 5 seconds, then an error message with a link to the CDN is displayed.

- **REQ-QRY-007** — Sparnatural is configured via the /api/sparnatural-config endpoint and executes queries against /api/sparql.

- **REQ-QRY-008** — The Sparnatural widget language is configured from the user preference in Settings.preferredLang, defaulting to 'en'.

- **REQ-QRY-009** — The Sparnatural widget is limited to 1000 results per query.

- **REQ-QRY-010** — The 'queryUpdated' and 'submit' events emitted by Sparnatural automatically trigger execution of the generated SPARQL query.
  - *If* If Sparnatural emits a queryUpdated or submit event with a non-empty queryString, then the query is sent to the backend and results are displayed.

- **REQ-QRY-011** — Sparnatural results display abbreviated URIs (local fragment after # or /) with the full URI as a tooltip.

- **REQ-QRY-012** — If a query execution error occurs in Sparnatural, a red error message is displayed in the results area.
  - *If* If the Sparnatural query returns an HTTP or network error, then the error message is displayed in red.

---

## 3. Gestion des requêtes SPARQL VizQ (CRUD)

- **REQ-QRY-013** — SPARQL VizQ queries are persisted in the browser's localStorage, separately per ontology.
  - *If* If the active ontology changes, then the displayed query list corresponds to that ontology's localStorage.

- **REQ-QRY-014** — The user can create a new query via the ➕ button in the list panel header.
  - *If* If the user clicks ➕, then a new query with an auto-incremented identifier (Query1, Query2…) is created, selected and saved.

- **REQ-QRY-015** — The auto-generated identifier of a new query avoids duplicates by incrementing until a free ID is found.

- **REQ-QRY-016** — The user can select an existing query by clicking its entry in the list.
  - *If* If the user clicks a query in the list, then its edit form is displayed in the right detail panel.

- **REQ-QRY-017** — The user can delete a query via the trash button shown on each list entry.
  - *If* If the user clicks the delete button of the selected query, then the query is removed from localStorage and the detail panel shows the empty state.

- **REQ-QRY-018** — Deleting a non-selected query does not change the current selection.

- **REQ-QRY-019** — The user can modify a query's identifier via the ID field in the form header. Spaces are automatically replaced by underscores.
  - *If* If the user types a space in the ID field, then it is immediately converted to an underscore.

- **REQ-QRY-020** — Changing a query's identifier updates the key in localStorage and refreshes the list.

- **REQ-QRY-021** — The user can define a label and a comment for each query.
  - *If* If a query has a label, then the label is displayed as the main text in the list and the ID appears as sub-text.

---

## 4. Editeur de patterns WHERE (triplets, filtres, optionnels)

- **REQ-QRY-022** — The user can add three types of patterns to the WHERE clause: Triple, Filter, Optional.

- **REQ-QRY-023** — A Triple pattern consists of a subject field, a predicate selector and an adaptive object field.
  - *If* If no triple is defined, then an invitation message is displayed instead of the pattern list.

- **REQ-QRY-024** — A triple's subject field offers autocomplete from variables already used in the query (datalist).

- **REQ-QRY-025** — The predicate selector is a custom icon-dropdown organized in groups: Classes (rdf:type), Object Properties, Datatype Properties, Annotation Properties.

- **REQ-QRY-026** — Properties in the predicate selector respect the subPropertyOf hierarchy and are displayed with indentation.

- **REQ-QRY-027** — When the predicate is rdf:type, the object field becomes a class tree dropdown (owl:Thing + hierarchy).
  - *If* If the predicate changes to rdf:type, then the object is reset to empty and replaced by a class tree selector.

- **REQ-QRY-028** — When the predicate is rdfs:label, rdfs:comment or a Datatype Property, the object field becomes a full-width text field accepting a variable or literal value.

- **REQ-QRY-029** — For other predicates (Object Properties, unknown), the object field is a short text field accepting a variable or IRI.

- **REQ-QRY-030** — When the predicate changes from rdf:type to another type, the object is reset to '?y'.

- **REQ-QRY-031** — A Filter pattern displays a monospace text field for entering a free FILTER expression.

- **REQ-QRY-032** — An Optional pattern contains its own sub-patterns (Triple and Filter) addable via internal buttons.
  - *If* If an OPTIONAL block is empty, then an 'OPTIONAL block vide' message is displayed inside it.

- **REQ-QRY-033** — Each pattern (triple, filter, optional) can be individually deleted via its trash button.
  - *If* If the user deletes a pattern, then the query is immediately saved and the form is reloaded.

- **REQ-QRY-034** — Any modification of a pattern field (subject, object, filter expression) triggers an immediate save and refreshes the SPARQL preview.

---

## 5. Options de requête (DISTINCT, ORDER BY, LIMIT)

- **REQ-QRY-035** — The user can enable the DISTINCT option via a checkbox.
  - *If* If DISTINCT is checked, then the DISTINCT keyword is inserted into the generated SELECT clause.

- **REQ-QRY-036** — The user can define a sort variable via the ORDER BY field, with autocomplete from query variables.

- **REQ-QRY-037** — The user can choose the sort direction ASC or DESC via a dropdown selector.

- **REQ-QRY-038** — The user can set the maximum number of results via the LIMIT field (default: 100, min: 1, max: 100000).
  - *If* If the LIMIT value is invalid, then the default value 100 is used.

---

## 6. Génération et aperçu SPARQL

- **REQ-QRY-039** — A preview of the generated SPARQL query is accessible via a collapsible 'SPARQL' panel at the bottom of the form.
  - *If* If the preview is hidden and the user clicks the SPARQL header, then the panel expands and the content is refreshed.

- **REQ-QRY-040** — The generated SPARQL query automatically includes the rdf, rdfs, owl prefixes and the current ontology's prefix.

- **REQ-QRY-041** — The SELECT clause contains all variables (starting with ?) found in triple subjects and objects. If no variable is detected, SELECT * is used.

- **REQ-QRY-042** — For a triple with a literal predicate and a non-variable non-quoted object, the SPARQL generation produces an intermediate variable and a FILTER(STR(?_lv) = "value") to handle language tags.

- **REQ-QRY-043** — Triples without a predicate are ignored during SPARQL generation.

- **REQ-QRY-044** — The SPARQL preview is automatically shown and refreshed when running a query if the preview was hidden.
  - *If* If the user clicks Run and the SPARQL preview is closed, then it opens automatically.

---

## 7. Exécution des requêtes et affichage des résultats

- **REQ-QRY-045** — The user can execute the query by clicking the '▶ Run' button.

- **REQ-QRY-046** — During execution, a status indicator 'Exécution…' is displayed.

- **REQ-QRY-047** — The query is sent as POST to /api/sparql with the SPARQL encoded in the body (Content-Type: application/x-www-form-urlencoded).

- **REQ-QRY-048** — After execution, the number of results is displayed in the status area.
  - *If* If the query returns N bindings, then the status displays 'N résultat(s)'.

- **REQ-QRY-049** — Results are displayed in an HTML table with alternating row colors (zebra striping).

- **REQ-QRY-050** — URI cells matching an internal entity (class, individual, property) are rendered as clickable links with a colored icon and the entity's display name.
  - *If* If a result URI matches a known internal entity, then a navigation link is displayed with the entity's icon and label.

- **REQ-QRY-051** — Clicking an internal entity in the results navigates to the corresponding application section and selects the entity.
  - *If* If the user clicks an internal URI in the results, then APP.navigate is called to the entity's section and the entity is selected after 150ms.

- **REQ-QRY-052** — External URIs (not recognized as internal entities) are displayed as hyperlinks opening a new tab.

- **REQ-QRY-053** — Literal values with a language tag (@fr, @en…) display the language as a superscript annotation.

- **REQ-QRY-054** — In case of an execution error, a red error message is displayed in the results area of the detail panel.
  - *If* If the query returns an HTTP error or network exception, then the error message is displayed in red and the status is cleared.

---

## 8. Recherche, liste et panneau redimensionnable

- **REQ-QRY-055** — A search field at the bottom of the list panel allows filtering queries by ID or label.
  - *If* If the user types text in the search field, then only queries whose ID or label contains that text (case-insensitive) are displayed.

- **REQ-QRY-056** — If no query matches the search, a 'No matching query' message is displayed.

- **REQ-QRY-057** — If no query is saved, a 'No saved query' message is displayed in the list.

- **REQ-QRY-058** — The list panel width is resizable by drag-and-drop via a split handle between the list panel and the detail panel.
  - *If* If the user drags the handle, then the list panel width is adjusted between 120px and 400px.

- **REQ-QRY-059** — During resizing, the CSS class 'resizing' is added to the body to disable text selection.

- **REQ-QRY-060** — The selected query is highlighted in the list with the CSS class 'selected'.

- **REQ-QRY-061** — When the SPARQL VizQ tab is reloaded, the previous selection is restored (restoreSelection).
  - *If* If _selectedId is set when the tab reloads, then the corresponding query is re-selected and its form re-displayed.
